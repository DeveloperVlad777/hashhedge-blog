# HANDOFF — конспект чата (перенос в Claude Code)

Краткая история того, как создавался этот проект. Полные правила — в `CLAUDE.md`,
эталонная спецификация — в `docs/article-generator-spec.json`.

## Что было сделано (по шагам)

1. **Исходные данные.**
   - Скрипт-генератор `hash_hedge_article_generator_strict_v20.json` (строгая
     структура HTML-блока статьи для блога Hash Hedge на Tilda).
   - Документ со статьёй (DOCX): одна статья «Стоп-лосс для проп-трейдеров: как
     не нарушить лимит дневного убытка» в трёх версиях — RU / EN / PT.

2. **Сгенерированы страницы.** По одной HTML на язык + общий `styles.css`
   (стили вынесены наружу по просьбе заказчика). Вёрстка — строго по спецификации:
   backlink, hook, title, meta, hero, кликабельное оглавление, тело, баннер по
   центру (inline-SVG логотип), FAQ-аккордеон, финальный промо-блок. Таблица
   рисков собрана из `<div>` (теги таблиц запрещены).

3. **Правка 1.** Добавлен заголовок к блоку FAQ; контейнеры изображений
   превращены в слоты под ссылку (`background-image: url('')` + `aria-label`).
   Inline-картинок доведено до двух (статья ≥1000 слов).

4. **Правка 2.** Заменён текст финального промо-блока (новый маркетинговый текст
   про #1 платформу, фондирование до $150К, вывод до 90% в USDT). Все CTA-кнопки
   переведены на ссылки регистрации со своим языковым кодом
   (`app.hashhedge.com/{lang}/register/`).

5. **Организация.** Заведена структура папок: общий `styles.css` в корне, статья —
   в подпапке `stop-loss-prop-traders/` с файлами `ru.html`, `en.html`, `pt.html`.

## Ключевые договорённости

- Кнопки «Начать челлендж» = регистрация (не логин).
- Для каждого языка — свой URL с языковым кодом.
- Стили только во внешнем `styles.css`, без дублей в HTML.
- Изображения — теги `<img>` в натуральную величину (ширина 100%, высота авто); ссылка в `src`.
- Логотип в баннере — только inline-SVG.

## Открытые вопросы / на будущее

- Язык kz (казахский) спецификацией поддерживается, но пока не сделан.
- Ссылки логина заведены в `CLAUDE.md`, но в кнопках пока не используются.
- Следующие статьи добавляются новой подпапкой по схеме из `CLAUDE.md` §3.

## Быстрый старт в Claude Code

1. Распакуй архив, открой папку проекта в Claude Code.
2. Claude Code автоматически прочитает `CLAUDE.md`.
3. Для новой статьи: «сделай статью <название> по правилам CLAUDE.md в новой
   подпапке, языки ru/en/pt» — приложи текст статьи.
